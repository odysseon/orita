# Good UI Design Principles

## 1. Colors

Three techniques for color harmony:

| Technique         | Description                                 | When to Use                       |
| ----------------- | ------------------------------------------- | --------------------------------- |
| **Monochromatic** | Pick one hue, vary lightness and saturation | Clean, minimal, professional look |
| **Complementary** | Choose two opposites for maximum contrast   | Call-to-action, emphasis, energy  |
| **Analogous**     | Three neighboring hues for harmony          | Calm, cohesive, natural feel      |

**Pro tip:** Use the 60-30-10 rule — 60% dominant color, 30% secondary, 10% accent.

---

## 2. Typography

| Principle            | Best Practice                                                                                                                 |
| -------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **Modular Scale**    | Don't pick random sizes. Use a fixed ratio (e.g., 1.25, 1.5, or the golden ratio 1.618). Each step is a multiple of the base. |
| **Visual Hierarchy** | Your eyes follow scale naturally. Larger = more important.                                                                    |
| **Font Discipline**  | One font family, 4 font sizes max.                                                                                            |

**Common Scale (1.25x — Major Third):**

- Base: 16px (body)
- Step 1: 20px (subheading)
- Step 2: 25px (heading)
- Step 3: 31.25px (display)

---

## 3. Spacing

| Rule                     | Example                                                               |
| ------------------------ | --------------------------------------------------------------------- |
| **One Base Unit**        | Pick one unit (e.g., 8px)                                             |
| **Stack Arithmetically** | 8, 16, 24, 32, 40, 48... (multiples of 8)                             |
| **Visual Rhythm**        | Consistent spacing = predictable, breathable layouts                  |
| **Border Radius**        | Use consistent radius values (e.g., 4px, 8px, 16px, 9999px for pills) |

---

## 4. Layout & Alignment

- **Grid System** — Use a 12-column grid for flexibility
- **Alignment** — Left-align text, center-align sparingly
- **White Space** — Let elements breathe; clutter kills readability
- **Proximity** — Group related items close together

---

## 5. Contrast & Accessibility

- **Text Contrast** — Minimum 4.5:1 for normal text, 3:1 for large text (WCAG AA)
- **Don't rely on color alone** — Use icons, labels, or patterns for meaning
- **Test with grayscale** — If your design works in B&W, it works everywhere

---

## 6. Consistency & Patterns

- **Reusable Components** — Buttons, cards, inputs should look and behave the same
- **Naming Conventions** — Use semantic names (e.g., `primary`, `danger`, not `blue`, `red`)
- **Design Tokens** — Store colors, spacing, fonts as variables for easy updates
